使用样例大概流程

使用样例大概流程

制作发布的指南，放坑必备

1/
创建gitbook文件夹
确认环境配置没有问题
检查
git --version
gitbook --version

写md文件，主要就是SUMMARY，README，从做好的文件夹复制过来就能懂。
整个结构写完以后，使用cmder，到gitbook文件夹里
gitbook init
gitbook serve

之后应该就生成了_book文件夹

2/
创建pub文件夹
去github网站上，自己的仓库里，新建一个仓库，只输入名字，注意”-“连接，不要大写，其他都不要设置，默认生成一个空仓，空仓的code页就有想要的网页地址。

注意你的ssh要设置好，本地生成一个ssh-key，并且把这个文件里的代码，copy到github的个人设置里。不设置就无法进行远程通信，参考链接

使用cmder，到pub文件夹里
git clone https://github.com/username/res-name.git
下载空仓库到pub文件夹，此时这个res-name文件夹里只有.git文件夹
进入.git文件夹找到config，里面添加 

[user]
    email = 90XXXXXX@example.com
    name = jine1212

res-name就是设置的空仓库的名字

把gitbook文件夹里的_book文件夹里面的文件copy出来，注意是里面的带index.html的，复制到pub文件夹/res-name文件夹里

git_bash执行下这个就可以了 git config --global --unset http.proxy 防止ssl报错，参考链接

使用cmder，到res-name文件夹里执行

git add --all
git commit -m "first commit"
git push -u origin master

一切顺利应该就让你输入github名称，和密码，输入github账号名和密码即可，然后就可顺利的上载。

如果不顺利，建议把pub文件夹整个删掉，然后照着2/ 重新做，并且把bug问题修复了。

3/
打开设置好的空仓库，此时应该有上载的文件，点开这个仓库的设置，不是系统设置，找到pages，添加来源，选择branch -master /boot 就行，不用设置主题，等这个界面生成的链接就是你的gitbook了。
